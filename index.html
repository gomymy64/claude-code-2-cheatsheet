01
Обнови Claude Code до 2.0 и подключи коннекторы

Настройка один раз — дальше пользуешься постоянно. Без этого шага ничего не заработает: агенты не смогут писать в Notion и Gmail.

  • Скачай или обнови приложение с claude.ai/download
  • Сверху в окне должна быть переключалка: Claude / Программирование / Совместная работа / Чат. Нет — значит старая версия, переустанови
  • Открой Customize в левом меню
  • Подключи Notion через OAuth в один клик
  • Подключи Gmail через OAuth в один клик
Настройки модели: внизу чата выбери модель (Claude 4.7) и контекстное окно. Для большинства задач хватает обычного режима мышления. Если лень подтверждать каждое действие — включай Bypass Permission.
02
Создай пустой GitHub-репозиторий за 30 секунд

Репо нужен чисто технически — Claude требует привязать его к каждому сценарию. Один раз создал — используй везде.

  • Открой github.com. Нет аккаунта — регаешься, бесплатно, пара минут
  • Правый верхний угол → кнопка «+» → «New repository»
  • Repository name: любое, например claude-agents
  • Оставь Public или Private — не важно для Claude
  • Поставь галку «Add a README file» (чтобы репо не был совсем пустым)
  • Жми «Create repository» — готово

Если ты не программист и слово «репозиторий» пугает — не парься. Ты просто создал технический контейнер, больше к нему не вернёшься. Claude использует его для внутренней логики.

03
Создай тестовую форму через Claude

Форма будет сквозным героем — сначала тренируемся с Preview, потом подключим к вебхуку в Шаге 6.

  • Открой новый чат в Claude Code
  • Вставь промпт ниже
  • Дождись пока Claude создаст форму в Preview-режиме справа
  • Держи этот чат открытым — вернёмся в Шаге 6
💬 Промпт 1 — Создание формы
Создай мне простую страницу с премиальным дизайном. Мне нужна форма с четырьмя полями: Имя, Email, Сколько у тебя денег, и Почему ты хочешь со мной работать. Сделай её красиво.
04
Агент по расписанию — «Ежедневный обзор»

Агент каждое утро будет собирать новости про ИИ и складывать в Notion. Один раз настроил — работает бесконечно.

  • В левой панели открой «Рутина» (Routines)
  • Жми «Новая рутина» → выбери «Удалённый» (Remote)
  • Добавь эмодзи 🧠 и название: «Ежедневный обзор»
  • Вставь промпт ниже
  • Выбери пустой GitHub-репо из Шага 2
  • Триггеры → «По расписанию» → выстави 8:00 ежедневно
  • Подключи коннектор Notion, остальные удали
  • Жми «Создать», потом «Запустить сейчас» (Run now) для теста
💬 Промпт 2 — Ежедневный обзор
Каждый день собирай последние новости про искусственный интеллект и автоматизации. Создавай короткую красивую страницу в Notion. Покажи мне 5 главных апдейтов по Claude Code и всё остальное, что стоит внимания. Максимум 8 пунктов.
Где смотреть результат: раздел «Запуски» (Runs) слева → кликаешь на последний → увидишь отчёт и ссылку на страницу в Notion.
05
Агент через вебхук — «Новый лид»

Тот же алгоритм, но триггер — внешний API-запрос. Заявки с формы / сайта / CRM прилетают, агент сразу обрабатывает.

  • «Рутина» → «Новая рутина» → «Удалённый»
  • Эмодзи 🎯 и название: «Новый лид»
  • Вставь промпт ниже
  • Выбери тот же GitHub-репо
  • Триггеры → «Вызов API» (API call / Webhook)
  • Claude выдаст уникальный URL и секретный токен — копируй оба, сохрани
  • Оставь только Notion и Gmail, остальные коннекторы удали (Fireflies тоже не нужен)
  • Жми «Создать»
💬 Промпт 3 — Обработка лидов (жёсткий приказ)
Каждый раз когда приходит новый лид через вебхук — выполняй три действия: 1. Зайди на сайт лида, изучи чем он занимается 2. Создай черновик приветственного письма на его email. Короткое, персональное, в моём стиле 3. Создай страницу в Notion с данными лида в виде таблицы: имя, email, бюджет, сайт, цель, мой ресёрч Важно: ничего не уточняй, вопросов мне не задавай. Получил вход - выполнил все три действия и завершил работу. В конце выдай одну строку-отчёт что сделано.

Секретный токен передаётся в заголовке запроса. Без него вебхук отклонит запрос — это защита от посторонних, без неё кто угодно мог бы дёргать твоего агента.

06
Подключи форму из Preview к вебхуку

Возвращаемся к чату с формой из Шага 3. Claude сам пропишет логику отправки — руками код трогать не надо.

  • Вернись в чат с формой (Шаг 3)
  • Скопируй URL вебхука из агента «Новый лид» (Шаг 5). Забыл — открой агента → нажми карандашик в настройках триггера
  • Вставь промпт ниже — подставь свой URL и токен
  • Заполни форму в Preview как обычный клиент: имя, email, бюджет, комментарий
  • Жми «Отправить»
  • Открой «Запуски» — увидишь новый с пометкой «Сработало через API»
  • Проверь: Notion → новая страница с лидом + Gmail → черновик в папке «Черновики»
💬 Промпт 4 — Подключение формы к вебхуку
Подключи эту форму к моему вебхуку. URL вот: [вставь URL]. Токен авторизации: [вставь токен]. При нажатии кнопки «Отправить» — POST-запрос на этот эндпоинт с данными из всех полей формы. Токен передавай в заголовке Authorization как Bearer.
07
Подключи вебхук к своему реальному сайту

Если у тебя уже есть свой сайт (Tilda, WordPress, Webflow, кастомный) — отправляй заявки напрямую туда, без Preview Claude.

⌨️ Код кнопки «Отправить» — JavaScript
const response = await fetch('ТВОЙ_URL_ВЕБХУКА', { method: 'POST', headers: { 'Authorization': 'Bearer ТВОЙ_ТОКЕН', 'Content-Type': 'application/json' }, body: JSON.stringify({ name: document.getElementById('name').value, email: document.getElementById('email').value, budget: document.getElementById('budget').value, message: document.getElementById('message').value }) });
Что подставить: ТВОЙ_URL_ВЕБХУКА — URL из агента «Новый лид». ТВОЙ_ТОКЕН — секретный токен из того же окна. ID полей (name, email, budget, message) — замени на ID полей твоей формы.

Для Tilda: в T123 (или любом блоке формы) → Настройки → Webhook → вставляешь URL, токен добавляешь в заголовки. Для WordPress/Webflow: через плагин WPForms или встроенный webhook-экшн. Плюс везде работает кастомный JS в HTML-блоке — код выше универсален.

Главное правило

Промпт = жёсткий приказ

Ошибка, из-за которой автоматизация ломается у всех

Если промпт написан «по-человечески» («помоги сделать…») — агент начнёт задавать уточняющие вопросы. А отвечать некому, он в облаке. Автоматизация зависнет.

  • Промпт для облачного агента — это ТЗ, не диалог
  • Структура: «получаешь X → делаешь A, B, C → возвращаешь отчёт»
  • Обязательная строчка: «Вопросов не задавай. Ничего не уточняй»
  • Если логика ветвится — пропиши все ветки в промпте явно

Перед тем как ставить агента на боевой режим — запусти вручную через «Run now» минимум 2 раза. Посмотри как он думает, допили промпт.

Ограничения

Чего агенты не умеют

Пропустишь этот блок — потратишь кучу нервов. Закладывай в сценарии заранее.

✕

Нет памяти между запусками

Каждый старт с чистого листа. Решил проблему клиента сегодня — завтра не вспомнит. Выход: прописывай «сохрани лог в Supabase / Notion» прямо в промпт.

✕

Нет доступа к локальным файлам

Агенты в облаке, твой Chrome / рабочий стол им недоступны. Выход: всё что нужно — грузи в Google Drive или Notion, давай API-доступ.

✕

Нет остановок в процессе

Агент не спросит «как тут поступить?». Любая неоднозначность в промпте → либо решает сам, либо зависает. Промпт должен быть однозначным.

✕

Недетерминированность

Это ИИ. 100 запусков = 100 немного разных ответов. Для строгих «если А → Б» бери n8n или Make — они инструменты-партнёры, не конкуренты.
